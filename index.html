##12.浅拷贝和深拷贝
浅拷贝：
只是拷贝了一份地址

但是复杂的数据类型，比如 var obj = {name:’lcb’,age:11,pets:[‘12’,’34’]};

##13.进程与线程
进程：程序的一次执行，已占有一片内存空间
线程：CPU的基本调度单位，是程序执行的一个完整流程

进程与线程的关系：1.一个进度至少有一个运行的线程：主线程；
2.一个进程中也可以同时运行多个线程，这时程序是多线程的；
3.一个进程内的数据可以供其中的多个线程直接共享；
4.多个进程之间的数据是不能共享的

浏览器运行时多进程的。
JS是单线程的。
优先执行初始化代码，遇到回调，则放到最后去执行。

关于为什么会造成外边距塌陷的分析

发表于 2019-08-22 | 分类于 css

大家应该会碰到这种情况。父元素内部有子元素，如果给子元素添加了margin-top，结果父元素也会一起跟下来，导致了外边距塌陷的现象。

我们可以在父元素中加入overflow:hidden，便可以解决外边距塌陷问题。

那为什么子元素加了margin-top，父元素也会跟下去呢？

根据规范，一个盒子如果没有上补白和上边框的话，那么它的上边距会和它的文档流中的第一个子元素的上边距重叠。意思就是子元素的margin-top如果找不到父元素的border或者padding，就会与父元素的上边距重叠。

其实，关于css动画，有几个点，我罗列了一下：transition、transform、translate和animation。

##1. transition
在一定时间内将属性从开始状态过渡到结束状态，但是它是要用事件去触发的，例如当鼠标滑过触发动画的开始。transition的属性可以给用户提供一种平滑过渡的动画，会让动画的效果好一些。

transition是有4个属性：
transition:transition-property transition-duration transition-timing-function transition-delay

1.transition-property:给指定css属性的过渡css状态值，比如设置为color，同时也可以设置为all，就是全部属性都将具有过渡效果。

2.transition-duration:过渡需要的时间，默认值是0。可以设置为1s/1000ms，来延长过渡的时间。

3.transition-timing-function:时间过渡函数，有几个参数可供选择：

4.transition-delay:过渡的延迟时间。默认值为0，即是没有时间延迟，立即开始过渡效果，单位也是s/ms。

IE垃圾回收器的变更

发表于 2019-08-21 | 分类于 JavaScript

IE的垃圾回收器是根据内存分配量搞的，变量的数量有个阈值（比如256个变量、4096个对象等等），达到这个阈值，才会运行垃圾回收机制。但是遇到一种情况，就比如说一个程序本身自带的变量就超过了这个阈值，那垃圾回收机制，就会频繁运行。性能就下降了。

IE7之后，他们对垃圾回收器的机制做了调整，讲阈值变为动态，如果垃圾收集例程回收的内存分配量低于15%，就将阈值加倍，如果回收了85%，就将阈值回复到默认值。这样提高了运行包含大量javascript的页面的性能。
